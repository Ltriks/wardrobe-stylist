---
name: wardrobe-product-planner
description: Plan a wardrobe management and outfit preview product into MVP phases, domain entities, pages, and implementation priorities.
---

When the user describes a wardrobe, closet, styling, or outfit product idea:

1. 用中文把需求整理成清晰的 product goal。
2. Identify:
- target user
- core use cases
- domain entities
- MVP scope
- later-stage scope
- technical risks
3. Propose phased implementation:
- Phase 1
- Phase 2
- Phase 3
4. Recommend the smallest buildable version first.
5. Produce acceptance criteria for the current phase.

Rules:
- Prefer realistic MVP over full vision planning.
- Treat AI try-on, recommendation engine, or generation-heavy workflow as later-stage unless explicitly prioritized.
- Prioritize data model clarity, usable workflow, and page structure.
- Keep the plan practical and ready for implementation.
